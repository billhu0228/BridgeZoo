## 0. 一句话定位

针对斜拉桥**正向悬臂拼装调索**这一**路径相关的混合整数非线性优化（MINLP）**问题，提出
"**精确仿射代理模型 + 两层（整数股数 / 连续张力）嵌套优化**"方法：用 `m+1` 次前进分析一次性
**精确**标定"张力→成桥应力/线形"的仿射映射，再在代理上以"LP 可行性相 + 解析梯度二次相"求解
内层，外层用应力引导的离散搜索调股数，并以真实有限元对代理做**在线监理校核**。

## 1. 创新性分析（论文卖点；本节是核心，供撰写正文与回应审稿）

写作原则：每个创新点写满三段——**(a) 本质是什么、(b) 与已有方法的区别、(c) 新颖性边界（诚实标注）**。
不夸大；把"经典思想的新用法"和"真正的新机制"分开讲，反而更稳、更经得起审。

### 创新点 1：贯穿分阶段施工全过程的**精确**仿射代理模型

- **(a) 本质**：固定索股 → 各索截面 `A` 固定 → `EA` 固定 → 每个施工阶段按**原始坐标**装配的整体
  刚度 `K` 与预张力无关；张力 `T` 只进入**荷载向量（RHS）**。因此对线性小位移前进分析，成桥终态的
  索应力 `σ` 与主梁线形误差 `err` 都是 `T` 的**精确仿射函数**：

  ```
  σ(T)   = σ0   + M · T      （M: m×m, 单位 MPa/N）
  err(T) = err0 + D · T      （D: d×m, 单位 m/N）
  ```

  标定只需 `m+1` 次前进分析：1 次 `T=0` 给常数项 `σ0/err0`，再逐索单位扰动（每次仅 `T[j]=1e6 N`）
  取差商得 `M、D` 的第 `j` 列。**线性后端下差商即真实斜率**，扰动步长只影响数值条件、不影响精度。
  （源码：`AffineCableModel` / `build_affine_model`，[linear.py:38-100](../bridgezoo/optim/linear.py)。
  现有测试 `test_affine_model_matches_fem`：随机张力下代理 vs FEM 误差 < 1e-6 MPa、< 1e-9 m。）

- **(b) 与影响矩阵法的区别（关键新颖性）**：经典"影响矩阵 / 最小二乘调索"通常建立在**成桥（完成态）
  结构**上，对单一线性体系做叠加。本文的 `M、D` 是**穿过路径相关、变刚度的整个前进分析过程**
  （梁段逐步激活、索逐步张拉、切线激活/位移锁定）后得到的端到端灵敏度，并且**同时**给出
  应力灵敏度 `M` 与线形灵敏度 `D`，统一支撑"应力带约束 + 线形目标"。论文需用一小段论证：
  **为什么路径相关过程下对 `T` 仍严格仿射**——因为 `T` 只出现在各阶段 RHS，`K_k` 与 `T` 无关，
  增量解 `Δu_k = K_k^{-1} ΔF_k(T)` 对 `T` 线性，逐阶段累加与终态应力恢复都是 `T` 的线性算子之和。

- **(c) 新颖性边界（必须诚实写）**：仿射性是线性叠加原理的结果，**数学上不新**；新颖在于
  (i) 证明并验证它**在分阶段、变刚度的施工全过程中仍严格成立**，(ii) 把它做成可直接驱动优化的
  **精确**代理（不是响应面/回归近似）。该精确性**只对线性小位移后端成立**；几何非线性
  （OpenSees `corotTruss`）下退化为局部近似，须给出 corot 交叉校核差异（量级：工作挠度
  ~0.01%、刻意放大的大挠度 ~1%，与 CLAUDE.md 工程约定一致）。

### 创新点 2：批量"同结构多右端（multi-RHS）"代理标定，实测 ~24× 提速

- **(a) 本质**：`m+1` 个标定工况**索股相同**，故各施工阶段的整体刚度 `K` **逐位相同**，仅 RHS
  （索张拉等效荷载）不同——这是典型的"同矩阵、多右端"。于是每个施工阶段**只装配并 `lu_factor`
  分解一次 `K`**，`m+1` 个右端一次性 `lu_solve` 回代，省掉 `m` 次冗余分解
  （`StagedDirectBatchSolver`，[direct.py:302+](../bridgezoo/fem/staged/direct.py)；
  数据布局由标量的 `{nid: (3,)}` 升为批量的 `{nid: (3, K)}`）。
- **(b) 与朴素实现的区别**：原 `build_affine_model` 逐工况跑完整 staged FEM，是整个优化的瓶颈
  （`n_seg=19` 时每次 ~4.7s、全程 ~20min）。改批量后**算法结果逐位不变**、只是去冗余。
- **实测（commit `add29bd`）**：`n_seg=19`（`m=38`，39 个右端）**4730 ms → 194 ms（24.4×）**；
  与逐列实现一致到浮点噪声（`max|dM|=1.8e-17 MPa/N`、`max|dσ0|=1.2e-10 MPa`），优化轨迹与最优解
  逐位不变（测试 `tests/test_staged_batch.py` 含 `balance_dofs`、随机非均匀张力、结构守卫）。
- **(c) 新颖性边界**：multi-RHS 复用 LU 分解是数值线性代数的**标准技巧**；新颖在于**识别出分阶段
  调索代理标定天然具备"同结构多荷载"结构**并据此重构求解器，把代理成本压到可嵌入外层离散搜索的
  量级——这是"两层框架在工程规模上可行"的关键支撑，而非纯算法噱头。

### 创新点 3：内层两相求解（LP 可行性相 + 解析梯度二次相），自带**可行性证书**

- **第一相 —— LP 可行性相**（[linear.py:133-149](../bridgezoo/optim/linear.py)）：

  ```
  min  s
  s.t. lower − s ≤ σ0 + M·T ≤ upper + s,   0 ≤ T ≤ T_max,   s ≥ 0
  ```

  解出的 `s*` 是**该股数配置下应力带的最小可达最大违反量**，是一个**可解释的结构可行性证书**：
  `s* ≈ 0` ⇒ 应力带 `[lower, upper]` MPa 可达，并顺带得到可行张力 warm-start；
  `s* > 0` ⇒ 这套股数**无论怎么张拉都进不了带内**，差距恰为 `s*` MPa。`T_max = tension_bound × A`
  （默认 1600 MPa × 面积）。该证书还能**反哺外层**判断"该不该改股数"。

- **第二相 —— 二次优化相（SLSQP + 解析梯度）**（[linear.py:152-190](../bridgezoo/optim/linear.py)）：
  在仿射代理上最小化完整目标（线形 + 应力均匀度 + hinge² 越界惩罚；股数项与 `T` 无关，是常数略去）。
  因代理仿射，目标对 `T` 的梯度可由 `M、D` **链式解析写出**（无有限差分，又快又稳）；应力带约束按
  第一相 `s*` 放宽（`band = s* + 1e-9`），**保证二次相最大违反不劣于 LP 最优**。

- **(b)(c) 解决的失效模式与边界**：把 FEM 直接塞进 SLSQP 当硬约束，一旦初始点不可行，优化器
  会"原地不动"而失败（旧 `FixedStrandTensionOptimizer` 路径，[continuous.py](../bridgezoo/optim/continuous.py)，
  现保留作非线性后端备用）。"先 LP 定可行域、再在精确线性模型上优化"彻底绕开此陷阱——这是相对
  "直接 MINLP / FEM-in-SLSQP"的明确优势。边界：两相的最优性依赖代理精确，故仅对线性后端成立。

### 创新点 4：两层（双层嵌套）混合整数优化框架

- **(a) 结构**：外层离散股数搜索 + 内层连续张力**精确**求解（`CableHybridOptimizer`，
  [hybrid.py](../bridgezoo/optim/hybrid.py)）。分解是**有原理的**：`M、D` 依赖截面 `A`，必须随股数
  重建 → 自然导出"外层定股数、内层定张力"的嵌套；外层只读内层返回的标量目标决定接受/拒绝。
- **外层手段**：
  (i) **`resize` 大步跳跃**（默认开）：利用 `σ ∝ N/A、力近似不变`，按 `σ_当前/σ_目标`（目标默认取
  应力带中值）一次性整体缩放股数 `N_new ≈ round(N_old × σ_当前/σ_目标)`，快速推到合理量级；
  (ii) **应力引导坐标搜索 ±step**（默认 step=1）：应力偏低先试减股（减面积→升应力）、偏高先试增股、
  带内优先减股省料；first-improvement 接受；
  (iii) 可选**随机重启**跳局部最优（默认 0）。
  默认外层迭代 4 轮，某轮 resize 与坐标搜索均无改进则提前停止。
- **跨层 warm-start**：改股数时按 `T_new = T_old × N_new/N_old`（保应力缩放）把旧张力传给内层做初值。
- **(c) 新颖性边界**：整数-连续双层分解是经典范式（不新）；新颖在于**内层是精确仿射子问题
  （白箱、非黑箱黑盒）**，使外层每个候选评估极廉价、且可借 `s*` 做结构可行性判断——把"混合整数
  难题"转化为"小规模离散搜索 + 廉价精确连续子问题"。

### 创新点 5：代理模型的"监理"——真实有限元在线校核（self-validating 设计）

- **(a) 机制**：在内层最优张力处用**真实有限元复评**，与仿射代理预测交叉校核；偏差
  `> 1 MPa`（`MODEL_MISMATCH_TOL_MPA`）即判定"后端实为非线性、代理不可信"，**fail-fast 报错**
  并提示改用 `slsqp` 后端（[linear.py:250-257](../bridgezoo/optim/linear.py)）。
- **(b) 论文表述**：把两个机制写成一对**互补关系**——"**仿射代理替昂贵 FEM 出力（保速度）**"
  与"**真实 FEM 反过来监理校核代理（保可信、自检适用域）**"。这是项目"一套定义、两个后端、
  结果可对照"（自研 direct vs OpenSees）一致性理念在优化层的延伸：代理给速度，监理给信任。
- **(c) 新颖性边界**：交叉校核思想常见；本文的贡献是把它做成**算法内置的适用域自检闸门**
  （越界即拒绝代理解、自动回退），而非事后人工核对。

## 2. 题目与摘要备选

### 2.1 题目备选
1. **（主推，方法侧重）** 基于精确仿射代理模型的斜拉桥分阶段施工调索两层优化方法
2. （力学/施工控制侧重）考虑正向悬臂拼装过程的斜拉桥索股—索力两层协同优化
3. （算法侧重）面向斜拉桥施工调索的仿射代理—两相—两层混合整数优化方法

### 2.2 摘要框架（中文 250~400 字）
- **背景**：斜拉桥悬臂拼装调索具有显著路径相关性，是整数股数 + 连续张力的 MINLP；传统经验试调
  与线性影响矩阵难以同时兼顾成桥线形、索力均匀与构造经济，直接把 FEM 嵌入优化器又易陷于不可行。
- **方法**：建立后端无关的二维正向施工力学模型；证明固定索股下"张力→成桥应力/线形"严格仿射，
  以 `m+1` 次前进分析**精确**标定代理，并以"同结构多右端"批量构造提速约 24 倍；内层用
  "LP 可行性相（给出可行性证书 `s*`）+ 解析梯度二次相"精确求解连续张力，外层用应力引导的离散
  搜索调股数，并以真实有限元对代理在线监理校核、越界即回退。
- **验证**：自研直接刚度法与 OpenSees 双后端在小位移阶段逐项吻合；与影响矩阵最小二乘、GA/PSO、
  直接 MINLP 等基线在成桥线形 RMSE、索应力均匀性、总股数与求解耗时上对比。
- **结论 + 边界**：方法在保证应力带可行的前提下显著提升求解效率与解的可解释性；精确性以线性
  小位移为适用域，几何非线性需走监理回退或非线性后端。

### 第 2 章 二维正向施工力学模型与前进分析

- 2.2 正向施工阶段定义：装梁段 → 激活并第一次张拉 → 合龙 → 二次调索 → 成桥评价。

- 3.2 `m+1` 标定与精度：差商即精确斜率；与 FEM 逐项一致（引测试数字）。
- 3.3 批量"同结构多右端"构造与提速：去冗余分解原理、数据布局、24.4× 实测与逐位一致性。
- 3.4 代理—FEM 交叉校核（监理）与适用域：`MODEL_MISMATCH_TOL_MPA`、fail-fast 回退。

### 第 4 章 两层优化框架
- 4.1 问题形式化：MINLP；设计变量（整数股数 + 连续张力，阶段优先排序）；

  权重默认 shape=1.0 / strands=0.02 / uniform=0.2 / violation=100，尺度 shape=1e-3 m、stress=100 MPa、
  strand=20）；约束 = 应力带 [800,1200] MPa（软约束）、张拉上限 1600 MPa、整数股数 [1,60]。
- 4.2 内层两相：LP 可行性相（`s*` 证书 + warm-start）→ 解析梯度 SLSQP 二次相（约束按 `s*` 放宽）。

## 5. 对比基线与消融（论证优势必需）

- **基线 1：影响矩阵最小二乘调索**（成桥态线性修正）——对照"路径相关 + 整数股数 + 多目标"处理能力。
- **基线 2：直接 MINLP / FEM-in-SLSQP**（即旧 `continuous.py` 路径）——对照鲁棒性（可行性陷阱）与速度。
- **基线 3：遗传算法 / 粒子群**——对照效率与解质量（多目标可但样本效率低）。
- **消融**：关 `resize`、关应力引导、退化为纯坐标搜索、批量 vs 逐列标定——论证各机制必要性。
  （`scripts/optimize_cables.py` 已提供 `--no-strand-resize`、`--no-stress-guided-strands` 等开关，便于直接做消融。）
