# 拉索股数与预张力优化算法说明

本文讲清楚 `scripts/optimize_cables.py` 这条命令背后到底在做什么，方便你学习和二次开发。
对应源码：

- 入口脚本：[scripts/optimize_cables.py](../scripts/optimize_cables.py)
- 算法实现：[bridgezoo/optim/](../bridgezoo/optim/) 包

---

## 1. 这个优化要解决什么问题

斜拉桥分阶段施工时，每个施工节段都要张拉一对拉索（右侧 + 左侧）。
对每一根索我们要决定两件事：

1. **股数 N**（整数）：这根索由多少根钢绞线组成 —— 决定了索的截面面积 `A = strand_area × N`。
2. **预张力 T**（连续实数，单位 N）：张拉时给这根索施加多大的力。

目标是：在满足应力约束的前提下，让**主梁成桥线形尽量贴合目标线形**，
同时**应力均匀、用料尽量省**。

这本质上是一个 **混合整数非线性优化（MINLP）**：

- 整数变量：股数向量（`2 × n_seg` 个）
- 连续变量：预张力向量（`2 × n_seg` 个）

变量排序统一为 **阶段优先**：`右1, 左1, 右2, 左2, ...`，
和分阶段建模器的输入格式一一对应（见 [variables.py](../bridgezoo/optim/variables.py) 的 `CableLayout`）。

---

## 2. 总体思路：把难问题拆成两层

直接同时优化"整数 + 连续"非常困难。本项目采用经典的**分层（嵌套）策略**：

```
外层：整数股数搜索（坐标搜索 + 启发式跳跃）
   └── 内层：给定股数后，求最优连续预张力
```

- **外层**负责"试探不同的股数配置"。
- **内层**负责"在股数固定后，把预张力调到最优"，并返回一个标量目标值。
- 外层只看内层返回的目标值好坏，决定接受或拒绝某个股数候选。

这套逻辑由 `CableHybridOptimizer` 统筹（[hybrid.py](../bridgezoo/optim/hybrid.py)）。

---

## 3. 目标函数：把多个诉求揉成一个数

要让优化器有方向，必须把"线形好不好、应力均不均、用料省不省、有没有超限"
压缩成**一个标量**。这在 [objectives.py](../bridgezoo/optim/objectives.py) 里完成：

```
目标 = w_shape   · (线形RMSE / 线形尺度)²        ← 线形贴合度
     + w_strands · (总股数 / 股数尺度)           ← 用料成本（线性项）
     + w_uniform · (应力标准差 / 应力尺度)²       ← 应力均匀度
     + w_viol    · (应力越界RMS / 应力尺度)²      ← 应力带违反惩罚
```

关键点：

- 每一项都先**除以一个"尺度"再平方**，让量纲不同的项（mm、根、MPa）变成可比的无量纲数。
- **应力越界惩罚 `w_viol` 默认权重 100，远大于其他项** —— 这是一个"软约束"：
  应力一旦跑出 `[stress_lower, stress_upper]`（默认 `[800, 1200]` MPa），
  惩罚会迅速增大，逼着优化器把应力拉回带内。
- 越界量用 **hinge（铰链）函数** 计算：带内为 0，带外才线性增长
  （见 `stress_violation_mpa`）。

权重和尺度都可以在命令行调（`--weight-*`、`--*-scale*`）。

### 一次"评估"是怎么算出来的

`CableDesignEvaluator.evaluate`（[evaluator.py](../bridgezoo/optim/evaluator.py)）是
连接"优化变量"和"结构物理"的桥梁，流程：

1. 把股数 / 预张力向量组装成分阶段施工模型（`build_staged_cantilever`）。
2. 调用结构求解器跑完整施工序列（`direct` 自写线性求解器，或 `opensees`）。
3. 取**成桥终态**：主梁各节点竖向位移 vs 目标线形 → 线形误差；各索终态应力。
4. 算出各项指标（RMSE、应力均值/标准差/min/max、越界量）和最终目标值。

## 4. 内层：给定股数，求最优预张力

这是整个算法**最精彩**的部分。有两种方法（`--continuous-method`）。

### 4.1 关键洞察：线性后端下，应力和线形是张力的"仿射函数"

`direct` 自写求解器（以及 OpenSees 的 `linear` Truss 模式）是**线性小变形**求解器。
这意味着：**固定股数后**，成桥终态的索应力 σ 和主梁线形误差 err，
都是预张力向量 T 的**仿射（线性 + 常数）函数**：

```
σ(T)   = σ0   + M · T      （M 单位 MPa/N）
err(T) = err0 + D · T      （D 单位 m/N）
```

既然是精确的线性关系，我们就**不需要在优化迭代里反复跑 FEM**，
而是先用少量 FEM 把这个线性模型"标定"出来，之后全在 numpy 里算。

**怎么标定（`build_affine_model`，[linear.py](../bridgezoo/optim/linear.py)）**：

- 先跑 1 次 `T = 0`，得到常数项 `σ0`、`err0`。
- 再对每根索逐个施加单位扰动（每次只让 `T[j] = 1e6 N`），跑 1 次 FEM，
  用差商 `(σ_j − σ0) / step` 得到矩阵 M 的第 j 列；D 同理。
- 共 `m + 1` 次 FEM（m = 索数）。对线性后端，这个差商就是**精确斜率**，
  扰动步长只影响数值条件，不影响精度。

### 4.2 两相求解（在仿射模型上，纯 numpy）

有了 `σ(T)`、`err(T)` 的解析式，连续子问题分两步：

**第一相 —— LP 可行性相**：先问一句"这套股数下，应力到底能不能压进带内？"

```
min  s
s.t. lower − s ≤ σ0 + M·T ≤ upper + s
     0 ≤ T ≤ T_max,   s ≥ 0
```

- `s` 是"应力带的最大违反量"，用线性规划（`scipy linprog`，HiGHS）求最小。
- 解出的 `s*`：
  - `s* ≈ 0` → 该股数配置下应力带**可达**，并顺带得到一个可行的张力起点（warm-start）。
  - `s* > 0` → 这套股数无论怎么张拉都做不到带内，差距就是 `s*`。
- `T_max` 是张拉时刻的张力上限：`tension_bound_stress × A`（默认 1600 MPa × 面积）。

**第二相 —— 二次优化相（SLSQP + 解析梯度）**：在可行的前提下，最小化完整目标。

- 目标 = 线形项 + 应力均匀项 + 越界惩罚项（股数项与 T 无关，是常数，略去）。
- 因为模型是仿射的，目标对 T 的**梯度可以解析写出**（见 `_model_objective_and_grad`），
  SLSQP 不用做有限差分，又快又稳。
- 应力带约束按第一相的 `s*` 放宽（`band = s* + 极小余量`），
  保证二次相的最大违反**不会比 LP 最优更差**。

**最后**：用真实求解器在最优张力处复评一次，并和仿射模型交叉校核。
若两者偏差 > 1 MPa（`MODEL_MISMATCH_TOL_MPA`），说明后端其实是非线性的
（比如 corot 大变形），仿射模型不可信，会报错并提示改用 `slsqp`。

> **为什么要这么绕？** 早期版本把 FEM 直接塞进 SLSQP 当约束，
> 一旦初始点不可行，硬约束会让优化器"原地不动"而失败。
> 先用 LP 找到可行域、再在精确线性模型上优化，彻底绕开了这个失效模式。

### 4.3 备用方法：`slsqp`（直接在 FEM 上优化）

`FixedStrandTensionOptimizer`（[continuous.py](../bridgezoo/optim/continuous.py)）是旧方法，
保留给**将来的非线性后端**：

- 直接用 SLSQP，目标和应力带约束都在每次迭代里调用真实 FEM（有限差分求梯度）。
- 同一个张力点的多次 FEM 调用用缓存共享，避免重复计算。
- 比 `linear` 慢得多，且有上面说的可行性陷阱，所以**默认不用**。

## 5. 外层：整数股数搜索

外层在 `CableHybridOptimizer.optimize` 里，目标是搜出更好的股数配置。
它综合了三种手段，逐步收敛：

### 5.1 初始解 + 随机重启（可选）

- 从命令行给的初始股数（默认每根 20 股）出发，先做一次内层优化，作为当前最优 `best`。
- 若 `--random-trials > 0`，再随机撒若干组股数，各做一次内层优化，
  谁更好就替换 `best`。这能跳出局部最优（默认 0，不开）。

### 5.2 每轮外层迭代的两个动作

循环 `--outer-iterations` 轮（默认 4），每轮做：

**动作 A —— resize 大步跳跃（默认开，`--no-strand-resize` 关闭）**

利用"应力 ∝ N/A、力近似不变"的近似，一次性把所有索的股数按
`σ_当前 / σ_目标` 比例整体缩放：

```
新股数 ≈ round(当前股数 × σ_当前 / σ_目标)
```

- 应力偏高的索 → 增股；应力偏低的索 → 减股；受压/松弛（σ≤0）的索 → 取最小股数。
- 目标应力默认取应力带中值（如 1000 MPa）。
- 这是一个"大步跳"，能快速把整体股数推到合理量级，避免一格一格地挪。

**动作 B —— 坐标搜索 ±step 精修**

逐根索尝试把股数 `+step` 或 `−step`（默认 step=1），每次只改一根：

- **应力引导**（默认开，`--no-stress-guided-strands` 关闭）：
  根据该索当前应力决定先试哪个方向 ——
  应力偏低先试**减股**（减面积→升应力），应力偏高先试**增股**，
  在带内则优先尝试减股省料。这让搜索更有的放矢。
- 每个候选都跑一次内层优化拿到目标值；
  **一旦有改进就立刻接受并跳到下一根索**（first-improvement 策略）。

**收敛判据**：某轮外层迭代里 resize 和坐标搜索都没带来改进，就提前停止。

### 5.3 warm-start 加速

改股数时，旧的预张力会按股数比例缩放后传给内层做初值
（`_scaled_pretension`：`T_new = T_old × N_new / N_old`），
让内层从一个接近的点出发，收敛更快。

---

## 6. 输出

最优解确认后，用真实求解器复评一次，写到 `--out` 目录（默认 `results/cable_opt/`）：

- `best_design.json`：逐索的股数、预张力、终态应力，以及各节点线形误差、汇总指标。
- `history.csv`：搜索过程中每次评估的目标值和指标（可画收敛曲线）。
- `summary.txt`：最优解的关键指标 + 应力带是否满足的结论。

命令行末尾还会打印**应力带判定**（`SATISFIED` / `VIOLATED`）和 LP 给出的理论下界 `s*`。
若加 `--verify-opensees`，会再用 OpenSees 后端独立复核一遍线形和应力。

```
                       ┌─────────────────────────────────────────┐
   初始股数 ──────────▶│  外层：整数股数搜索 (CableHybridOptimizer) │
                       │                                           │
                       │   随机重启(可选)                          │
                       │   每轮外层迭代:                           │
                       │     A. resize 按应力比例整体改股(大步跳)  │
                       │     B. 坐标搜索 ±step 逐索精修(应力引导)  │
                       │     接受/拒绝 ← 看内层返回的目标值         │
                       └───────────────┬───────────────────────────┘
                                       │  给定一组股数
                                       ▼
                       ┌─────────────────────────────────────────┐
                       │  内层：连续预张力优化 (method=linear)      │
                       │                                           │
                       │   1. 标定仿射模型 σ=σ0+M·T, err=err0+D·T  │
                       │      (m+1 次真实 FEM)                     │
                       │   2. LP 可行性相: min s → s*, warm-start   │
                       │   3. SLSQP 二次相: 解析梯度最小化总目标    │
                       │   4. 真实 FEM 复评 + 交叉校核              │
                       └───────────────┬───────────────────────────┘
                                       │  返回最优张力 + 标量目标值
                                       ▼
                            评估目标(线形+用料+均匀+越界惩罚)
```

## 8. 常用命令

```bash
# 基本运行（n=4 个节段，4 轮外层迭代，2 次随机重启）
py -3.12 -m scripts.optimize_cables --n 4 --outer-iterations 3 --random-trials 2

# 调整应力带和权重
py -3.12 -m scripts.optimize_cables --n 6 --stress-lower 850 --stress-upper 1150 \
    --weight-shape 1.0 --weight-strands 0.05

# 关掉启发式，退化成纯坐标搜索（教学对比用）
py -3.12 -m scripts.optimize_cables --n 4 --no-strand-resize --no-stress-guided-strands

# 用 OpenSees 独立复核结果
py -3.12 -m scripts.optimize_cables --n 4 --verify-opensees
```

## 9. 学习路线建议

1. 先读 [objectives.py](../bridgezoo/optim/objectives.py)（最短）—— 搞清"什么是好解"。
2. 再读 [evaluator.py](../bridgezoo/optim/evaluator.py) 的 `evaluate` —— 看一次评估怎么从
   变量算到指标。
3. 重点啃 [linear.py](../bridgezoo/optim/linear.py) —— 仿射模型 + 两相求解，是全篇精华。
4. 最后读 [hybrid.py](../bridgezoo/optim/hybrid.py) 的 `optimize` —— 外层如何调度内层。
5. `continuous.py` 作为对比阅读，理解"为什么不直接把 FEM 塞进优化器"。
